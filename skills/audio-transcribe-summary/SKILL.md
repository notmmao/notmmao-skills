# 音视频转录总结技能

## Overview
本技能整合了音视频下载、语音识别、内容总结全流程能力，无需手动操作多个工具，仅需提供原始资源即可自动完成从音视频到结构化文本输出的全链路处理。

## 执行工作流

### 步骤1：资源识别与音频提取
- **若为B站/YouTube/抖音等主流视频平台URL**：调用内置`yt-audio-download`技能，基于yt-dlp工具自动下载视频音轨，默认转换为高质量MP3格式保存，支持直接URL输入或模糊搜索视频标题匹配下载
- **若为本地音视频文件**：直接读取本地文件，提取音频流用于后续处理

### 步骤2：音频转文字（ASR）
优先使用`本地whisper`的small模型转换，语言优先使用简体中文，支持中、英、日、韩等多语言识别.

### 步骤3：内容结构化处理
根据用户需求对转写文本进行定制化处理，支持输出格式包括但不限于：
- 核心观点摘要
- 逻辑脉络梳理
- 重点内容提炼
- 行动要点清单
- 思维导图结构
- 问答整理格式
处理完成后自动输出markdown格式文件留档

### 步骤4：默认导出PDF（可选）
处理完成后自动调用`lovstudio-any2pdf`技能，使用Classic Thesis风格从markdown文件转换为PDF文件，若用户未指定输出格式，**默认自动将结构化总结内容导出为PDF文件**，PDF需包含清晰的排版、标题层级，便于用户阅读和保存。

### 步骤5: 更新本地支持库

本地知识库在 `<workspace>/my-knowledge-base`
更新流程: 
1. 创建分类文件夹
2. 更新 index.md 文件, 模板如下
```markdown
## 军事类
### 美伊冲突相关
- [美伊冲突美军进攻性武器汇总](./军事类/美伊冲突美军进攻性武器汇总.md) | [PDF版本](./军事类/美伊冲突美军进攻性武器汇总.pdf)
```
3. 更新 log.md 文件
```markdown
## 2026年4月13日
- 新增内容：《美伊冲突美军进攻性武器汇总》，来源B站视频[《在美伊冲突之中，美国使用了哪些进攻型武器？（60分钟特辑）》](https://b.tv/bvid9527)，包含markdown和pdf两个版本
```

## 输出要求
- 总结内容需忠实于原始音视频内容，不得添加无关信息
- 根据用户需求调整总结长度（简明摘要/详细摘要/全文转写）
- 结构化输出优先使用层级清晰的列表格式，方便阅读
- 若输入资源为视频平台链接，总结内容需在开头显著位置标注原始视频链接，方便用户溯源查看原内容
- **默认输出规则**：用户未明确指定输出格式时，优先提供PDF格式的总结文件，同时可附加纯文本版本供用户快速查看
- 每一级处理步骤都要有缓存，同一个输入URL优先从缓存中获取已处理结果，避免重复执行任务
- `log.md` 记录任务更新信息

### 输出目录结构

```
<cache>/<skill-name>/
├── log.md                 # 任务信息
└── <video-title>/
    ├── video.mp4          # Original video
    ├── audio.mp3          # Extracted audio
    ├── transcript.txt     # ASR 转录的文本
    ├── summary.md         # Structured summary
    └── summary-{style}.pdf        # Structured summary pdf 版本
```
